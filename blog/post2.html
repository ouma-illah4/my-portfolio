My First HTML & CSS Project: What I Built and What I Learned

August 22, 2025 | Category: Tech

I created a simple, responsive portfolio using plain HTML and CSS. The goal was to learn clean structure, mobile-first design, and basic components like a header, project cards, a blog section, and a contact form.

Tools I Used

  • VS Code (with Live Server)
  • A single style.css for global styles
  • Semantic HTML elements (header, section, main, footer)

Basic File Structure

.
├── index.html
├── blog.html
├── post1.html
├── post2.html   <-- this page
├── style.css
└── image/
    ├── ILLAH.jpg

Key Layout Pattern (Card + Grid)

Project and blog cards use a simple grid so they adapt to screen size.

/* Grid shell */
.container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 20px;
}

/* Card */
.card {
  background: #fff;
  border-radius: 10px;
  box-shadow: 0 4px 6px rgba(0,0,0,.1);
  padding: 20px;
}

Mobile-First Tip

Start with a single-column layout, then enhance with media queries: 

@media (min-width: 768px) {
  .nav { display: flex; justify-content: center; }
  .container { grid-template-columns: repeat(3, 1fr); }
}

What I Learned

  • Keep styles modular: sections get their own blocks in style.css.
  • Use IDs for section anchors (e.g., #projects, #contact).
  • Test on mobile sizes early, not at the end.

Next steps: add more pages (like this post), improve accessibility (labels, contrast), and eventually introduce a lightweight CSS framework or utility classes.
